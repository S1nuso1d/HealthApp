Пользователи все чаще используют мобильные приложения, фитнес-браслеты и носимые устройства для отслеживания физической активности, сна, питания, уровня гидратации и других показателей здоровья. Однако большинство существующих решений обладают ограниченной аналитикой и не позволяют формировать комплексную картину состояния пользователя.Актуальность разработки системы HealthApp обусловлена необходимостью объединения различных категорий данных пользователя в единую интеллектуальную платформу, способную не только хранить показатели, но и выполнять их анализ с последующим формированием персональных рекомендаций.В ходе прохождения производственной преддипломной практики была разработана информационная система HealthApp, представляющая собой мобильную экосистему анализа данных здоровья пользователя.Целью работы являлась разработка полнофункционального программного комплекса, включающего:мобильное Android-приложение; серверную часть; систему хранения данных; аналитический модуль; AI-модуль рекомендаций. Для достижения поставленной цели были решены следующие задачи:анализ предметной области; проектирование архитектуры системы; разработка структуры базы данных; реализация backend-сервиса; разработка Android-приложения; реализация аналитических алгоритмов; разработка AI-модуля; тестирование системы. Практическая значимость работы заключается в создании программного комплекса, позволяющего пользователю получать комплексную аналитику состояния здоровья на основе совокупности различных факторов.[МЕСТО ДЛЯ РИСУНКА]Рисунок 1 – Общая концепция системы HealthApp2. Архитектура и технологический стек системыСистема HealthApp реализована по клиент-серверной архитектуре.Основными компонентами являются:Android-клиент; backend-сервер; база данных; аналитический модуль; AI-модуль. Мобильное приложение взаимодействует с backend через REST API. Серверная часть выполняет обработку запросов, хранение данных, аналитику и генерацию рекомендаций.[МЕСТО ДЛЯ СХЕМЫ]Рисунок 2 – Архитектура системы HealthApp2.1 Технологический стекДля реализации проекта использовались современные технологии разработки программного обеспечения.Android-клиентKotlin; Jetpack Compose; MVVM; Hilt; Retrofit; Navigation Compose; Coroutines; DataStore. BackendPython; FastAPI; SQLAlchemy; Pydantic; JWT; Uvicorn. База данныхPostgreSQL; SQLite. АналитикаPandas; NumPy; Scikit-learn. AI-модульOllama; локальные LLM; recommendation engine. 2.2 Взаимодействие компонентовПри запуске приложения пользователь проходит авторизацию, после чего мобильный клиент получает данные профиля и аналитики через API.Основные этапы обработки:Пользователь вводит данные; Данные отправляются на backend; Backend сохраняет информацию в БД; Аналитический модуль обрабатывает показатели; AI-модуль формирует рекомендации; Результаты отображаются в интерфейсе приложения. [МЕСТО ДЛЯ БЛОК-СХЕМЫ]Рисунок 3 – Схема взаимодействия компонентовТаблица 1 – Используемые технологииКомпонентТехнологияНазначениеMobileKotlin + ComposeAndroid UIBackendFastAPIREST APIORMSQLAlchemyРабота с БДБДPostgreSQLХранение данныхАвторизацияJWTБезопасностьСетьRetrofitHTTP-запросыDIHiltВнедрение зависимостейAIOllamaГенерация рекомендаций3.
